# Privacy and Feedback

Feedback categories are **Bug**, **Experience**, and **Quality**. Include only
the package version, platform/version, fictional scenario, expected behavior,
and a sanitized description.

Do not submit a complete resume, job description, name, phone number, email,
address, identity number, real employer/client confidential information,
unpublished vacancy content, API key, access token, password, credential, or
private Agent conversation. Reproduce issues with the fictional samples. If a
personal example is essential, redact it and retain only the minimum fragment.

This kit stores and transmits nothing by itself.

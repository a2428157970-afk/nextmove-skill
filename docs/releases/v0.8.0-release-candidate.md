# NextMove v0.8.0 Release Candidate

Date: 2026-07-13
Status: Ready for human release approval; not tagged or published

## Version scope

This candidate integrates Epic-025.3 through Epic-026.2 on top of `main`.
It contains evidence-based match explanation, the Career Intelligence
Benchmark, expanded career taxonomy, career-transition intelligence, the
Human Career Report, and separate Runtime and Prompt-only distribution modes.
The Skill version remains `0.8.0`; `skill/__version__.py` is the only code
source for package version metadata.

No product feature, scoring rule, public API, runtime dependency, backend, or
frontend change was added during release-candidate finalization.

## Core capabilities and contracts

- Resume analysis and structured profile extraction.
- Resume improvement, job matching, career advice, and fixed-order
  `career_analysis` through `NextMoveSkill.run()`.
- Evidence-based matching and internal career-stage/transition intelligence.
- Six-field `JobMatchResult`: `match_score`, `matched_skills`,
  `missing_skills`, `strengths`, `gaps`, and `recommendations`.
- Provider-neutral Skill Core and application-to-public-Skill dependency
  direction.
- Unchanged Agent, Application, CLI, and public serialization contracts.

## Human Career Report

The Runtime Package includes a deterministic Human Career Report demonstration
for fictional HR, technology, and Sales-to-Product-Manager scenarios. The
reporting model remains an internal boundary and is not added to existing
Agent, Application, CLI, or Skill response schemas.

## Distribution and installation boundaries

### Runtime Package

`nextmove-skill-package-0.8.0.zip` contains the Python 3.11+ Skill runtime,
manifest, checksums, examples, Quick Start, real offline preflight, and report
demonstration. It has no third-party runtime dependencies. Extract the complete
archive and run the preflight before importing or modifying its contents. Only
`NEXTMOVE_READY` proves runtime readiness.

### Prompt-only Kit

`nextmove-prompt-kit-0.8.0.zip` contains reference files and prompts, but no
Python implementation. Its preflight result is `NEXTMOVE_PROMPT_ONLY` and must
never be presented as runtime execution.

Ordinary ChatGPT file upload is Prompt-only use. It cannot execute the local
NextMove Python Runtime. A separate code-enabled integration would require
actual host-specific verification.

## Privacy

All bundled examples are fictional and contain no contact details. The build
and validation flow does not retain resume, job-description, identity,
employer, credential, or private conversation content. Do not replace bundled
examples with personal data for release evidence.

## Known limitations

- ChatGPT ordinary uploads are Prompt-only and do not run local Python.
- Claude Code, Cursor, and ChatGPT external-host runtime status remains Pending.
- The Runtime requires Python 3.11+ and local filesystem access.
- The strict directory preflight should run before imports or edits: generated
  files such as `__pycache__` are correctly treated as a manifest mismatch.
- Human Career Report types remain internal and are not public API contracts.
- Optional live AI transports remain externally assembled and opt-in; this
  release candidate performs no network call and bundles no credential.
- ZIP download behavior and published checksum display still require human
  verification after release assets are uploaded.

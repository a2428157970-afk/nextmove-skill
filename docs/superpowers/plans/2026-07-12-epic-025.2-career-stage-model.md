# Epic-025.2 Career Stage Model Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add deterministic, evidence-based Career Stage assessment to resume analysis and career advice without changing any public Skill or Agent contract.

**Architecture:** Keep the current analysis and advice public result types unchanged. Add internal stage schemas, taxonomy, signal extraction, and assessment inside `skill/career/`; map the result to the existing legacy `career_level`; and use the same assessment to tailor advice. All processing remains local, deterministic, provider-neutral, and dependency-free.

**Tech Stack:** Python 3.11 standard library, dataclasses, enums, regular expressions, unittest.

## Global Constraints

- Do not modify Skill API signatures, `NextMoveSkill.run()`, `JobMatchResult`, Application Layer, Agent Contract, CLI, release files, or Git tags.
- Do not modify `skill/interface.py`, `skill/agent/`, `skill/matching/`, `skill/ai/application/`, backend, frontend, provider code, database code, packaging, or dependencies.
- Preserve application -> public Skill dependency direction and keep all new logic inside Skill Core.
- Keep `ResumeAnalysisResult.career_level` and `CareerAdviceResult.current_level` public names, types, defaults, serialization keys, and permitted legacy values unchanged.
- Do not add public `career_stage`, confidence, signal, score, or breakdown fields.
- Use TDD: each new production behavior requires a failing test first, then minimal implementation and a passing rerun.

## File map

- Create `skill/career/stages.py`: internal `CareerStage`, `StageConfidence`, `CareerStageAssessment`, and legacy mapping.
- Create `skill/career/stage_assessor.py`: normalized evidence extraction, deterministic stage assessment, confidence, and date-duration handling.
- Modify `skill/career/advisor.py`: consume an assessment for stage-specific paths, gaps, and actions while returning legacy public `current_level`.
- Modify `skill/analysis/analyzer.py`: obtain assessment and map it to existing `career_level`.
- Create `tests/test_career_stage_assessor.py`: taxonomy, assessment, confidence, duration, and anti-inflation tests.
- Modify `tests/test_resume_analyzer.py`: analyzer mapping and existing behavior regressions.
- Modify `tests/test_career_advisor.py`: stage-specific advice and career-change behavior.
- Modify `tests/test_skill_serialization.py` and `tests/test_agent_tools.py`: exact public-key and Agent enum regressions.
- Modify `CHANGELOG.md`, `PROJECT.md`, and `NEXT_TASK.md` only after acceptance.

# Skill Distribution Package Implementation Plan

> **For agentic workers:** Follow this plan in small, test-first slices. Keep the
> Skill Core, Agent Contract, Application Layer, CLI, providers, dependencies,
> backend, and frontend unchanged.

**Goal:** Produce a versioned, self-contained NextMove Skill distribution
archive that a non-Python user can load, verify, and use for a first fictional
Human Career Report in less than five minutes.

**Architecture:** Add distribution-owned source documents and fictional sample
assets, then assemble them with byte-for-byte canonical runtime files into a
temporary `nextmove-skill-package/` directory and ZIP archive. Validation checks
structure, versions, links, prompts, privacy, and the adoption flow. The package
is never imported by Skill Core and does not alter existing runtime behavior.

**Constraints:** Standard library only. No new dependency, network service,
user system, platform adapter, runtime schema, or CLI behavior.

- [ ] **Step 1: Add package-local link checks**

Extract Markdown links with a bounded standard-library parser. For each relative
link, resolve it against the source file and require an existing in-package
target. Reject `file://`, absolute local paths, repository-parent paths, and
links to files omitted from the manifest. Permit only `https://` for explicitly
labelled external documentation and feedback links.

- [ ] **Step 2: Add agent-status consistency checks**

Require every platform row to state all applicable verification levels and
prohibit unsupported words such as `guaranteed`, `fully supported`, or
`verified` without a dated evidence record. Specifically require ChatGPT
guidance to say that upload/knowledge access does not execute NextMove.

- [ ] **Step 3: Add feedback and privacy checks**

Require Bug, Experience, and Quality categories plus a warning before any
free-text feedback template. Require the warning to prohibit complete resumes,
names, phone numbers, email addresses, company/client secrets, unpublished job
content, API keys, and tokens. Require fictional-sample reproduction as the
preferred route.

- [ ] **Step 4: Add forbidden-language scans**

Scan user-facing guidance and prompts for employment guarantees, deterministic
success/failure, unsuitable verdicts, and instructions to fabricate missing
details. Exclude quoted negative examples only when the surrounding instruction
explicitly prohibits them.

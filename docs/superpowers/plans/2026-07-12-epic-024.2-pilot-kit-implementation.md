# Epic-024.2 Pilot Kit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver an Agent-first Pilot Kit that lets a non-technical job seeker complete a fictional NextMove career analysis in five minutes, understand the result, and submit privacy-safe feedback.

**Architecture:** Keep the Skill and CLI unchanged. Add a documentation-only adoption layer around existing `NextMoveSkill.run`: fictional UTF-8 inputs, copyable Agent prompts, a report-reading guide, dual feedback templates, and focused repository acceptance tests that execute the real sample through the existing Skill API.

**Tech Stack:** Markdown, UTF-8 text fixtures, Python 3.11 standard library, unittest, GitHub Issue templates.

## Global Constraints

- Do not modify `skill/`, `application/`, backend, frontend, AI Provider, CLI behavior, `CareerAnalysisReport`, or public API contracts.
- Do not add dependencies, Web infrastructure, databases, authentication, telemetry, or network requirements.
- Sample data must be entirely fictional, public-safe, and free of contact information, real employers, company secrets, and credentials.
- Agent prompts must prohibit fabricated experience, skills, education, projects, certifications, achievements, employers, and metrics.
- Anonymous feedback is the primary channel; GitHub Issue is secondary.
- Do not collect resume bodies, job-description bodies, identity, contact details, company information, credentials, or raw private Agent conversations.
- Do not implement `--job-description-file`, a CLI summary mode, or CLI error-format changes.
- Do not create a Git tag, push, or GitHub Release.

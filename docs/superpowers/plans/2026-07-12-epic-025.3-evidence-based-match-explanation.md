# Epic-025.3 Evidence-based Match Explanation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an internal deterministic requirement-evidence explanation layer to job matching while preserving the exact public `JobMatchResult` and Agent output contract.

**Architecture:** Reuse the Epic-025.1 domain-aware scorer as the sole score owner. Add internal requirement assessments and an explanation builder inside `skill/matching/`, then have `JobMatcher` translate the per-call `MatchExplanationResult` into six legacy fields. No explanation object crosses the public Skill boundary.

**Tech Stack:** Python standard library, dataclasses, enums, regular expressions, and `unittest`; no new dependency.

## Global Constraints

- Do not modify `NextMoveSkill.run()`, `skill/interface.py`, `JobMatchResult`, Application Layer, `skill/agent/`, CLI, providers, database code, dependency manifests, frontend, backend, release files, or Git tags.
- Preserve exactly these result fields and serialized keys: `match_score`, `matched_skills`, `missing_skills`, `strengths`, `gaps`, `recommendations`.
- Keep all explanation types internal to `skill/matching/`; do not export them through `skill.schemas`, Agent schemas, public method signatures, or result serialization.
- Use resume-provided professional evidence only. Exclude contact data and never invent skills, credentials, responsibilities, scope, metrics, or experience.
- Non-mention is `unknown`, never `missing`. `missing` requires explicit negative evidence or an explicit qualification conflict.
- Use TDD: test red, minimal code, test green, then a focused commit.
- Before coding, confirm the completed Epic-025.1 implementation exists. Do not recreate it in this Epic.

## File Map

- Modify `skill/matching/schemas.py`: internal explanation enums/dataclasses; preserve `JobMatchResult`.
- Modify `skill/matching/scoring.py`: add requirement-level evidence to the Epic-025.1 assessment.
- Create `skill/matching/explanations.py`: deterministic strength/gap/risk derivation.
- Modify `skill/matching/matcher.py`: internal composition and legacy-field mapping.
- Create `tests/test_matching_explanations.py`: evidence semantics and four scenario fixtures.
- Modify `tests/test_matching_scoring.py`, `tests/test_job_matcher.py`, `tests/test_skill_serialization.py`, and `tests/test_agent_tools.py`.
- Modify `CHANGELOG.md`, `PROJECT.md`, and `NEXT_TASK.md` only after acceptance.

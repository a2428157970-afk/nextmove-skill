# Epic-025.1 Domain-aware Job Matching Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the technical-only keyword matcher with deterministic, domain-aware job matching while preserving the existing Skill API and Agent Contract.

**Architecture:** Keep `JobMatcher` as the public facade. Add internal taxonomy, classification, and scoring units under `skill/matching/`, then map their assessment into the unchanged six-field `JobMatchResult`. All behavior is offline, rule-based, provider-neutral, and dependency-free.

**Tech Stack:** Python 3.11 standard library, dataclasses, enums, regular expressions, unittest.

## Global Constraints

- Do not modify the Skill API, `NextMoveSkill.run()`, `NextMoveSkill.match_job()`, Application Layer, or Agent Contract.
- Do not modify `skill/interface.py`, `skill/agent/`, `skill/ai/application/`, backend, frontend, Web, provider, database, CLI, packaging, dependency, or `v0.8.0` tag state.
- Do not add an AI Provider, database, Web/Frontend code, or new dependency.
- Preserve application -> Skill dependency direction; `skill/` must not import an Application Layer.
- Keep all six `JobMatchResult` fields, types, defaults, and serialized keys unchanged.
- Use TDD and make no broad refactor outside `skill/matching/`.

## File map

- Create `skill/matching/taxonomy.py`: v1 enums, families, aliases, adjacent domains, and fixed evidence weights.
- Create `skill/matching/classifier.py`: input normalization, evidence collection, and domain/family classification.
- Create `skill/matching/scoring.py`: requirement extraction, qualification comparison, weighted components, confidence, and assessment.
- Modify `skill/matching/schemas.py`: add internal frozen schemas while preserving `JobMatchResult` exactly.
- Modify `skill/matching/matcher.py`: orchestrate the internal pipeline and map to the legacy result.
- Modify `tests/test_job_matcher.py`: public HR/technology/finance/operations/insufficient-information behavior and legacy regression tests.
- Create `tests/test_matching_classifier.py`: taxonomy and classification unit tests.
- Create `tests/test_matching_scoring.py`: component, weighting, confidence, and edge-case tests.
- Modify `tests/test_skill_serialization.py`: exact six-key compatibility assertion.
- Modify `tests/test_application_workflow.py`: only the existing dependency-boundary assertion if needed to cover new matching files.
- Modify `CHANGELOG.md`, `PROJECT.md`, and `NEXT_TASK.md` only after implementation acceptance.
